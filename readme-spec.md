## 現状と課題

- 36.3.129.234 に Barix Instreamer があり、163.43.157.215（sakuraサーバ）で受信・配信を行っている。
- 現在は icecast を利用しているが、同じ音声が1時間遅れで繰り返し配信される現象が発生している。
- この問題を解決するため、新たな配信システムを構築する。

## 目的

- Instreamer からの音声をリアルタイムで受信し、Web上でライブ配信を行う。
- 同時に、指定した遅延（初期値30分）を設けた音声もWebで配信可能とする。

## 構成案

1. Instreamer → Ruby（TCP受信、ffmpegで処理）
2. Ruby（Sinatra）→ Web公開（ライブ配信）
3. Ruby（Sinatra）→ Web公開（x分遅延配信、初期値30分）
